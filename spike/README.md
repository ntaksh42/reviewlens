# ReviewLens Impact Spike

差別化機能の**価値検証だけ**を目的とした使い捨て拡張。PR配管（認証・PR一覧・diff表示・コメント）は一切作らない。

## 検証したい問い
> 「変更された関数の *未変更の呼び出し元* を自動で炙り出す」は、実際のレビューで便利か？

## 動かし方
1. この `spike` フォルダを VS Code で開く
2. `F5`（Run Impact Spike）→ Extension Development Host が開く
3. その新ウィンドウで**検証したい実リポジトリ**を開く（TypeScript / C# / Python など Call Hierarchy が強い言語が望ましい）
4. 設定 `reviewlens.baseRef` を比較したいブランチに（既定 `main`）
5. アクティビティバーの **ReviewLens（望遠鏡アイコン）→ Impact** ビューを開き、タイトルの検索アイコンで解析実行

`git diff <baseRef>...HEAD` が空なら、作業ツリーの変更 → 全変更の順にフォールバックする。

## 見るべきもの（成功基準）
- Impact ツリーに「変更された関数」と「その呼び出し元」が出るか
- `⚠`（未変更の呼び出し元）が、レビュアーとして**実際に確認したくなる箇所**か
- ノイズが多すぎないか / 体感速度は許容できるか

→ `⚠` が「diffだけ見ていたら見落とす波及」を繰り返し当てるなら**差別化は実証**。次は SPEC §19.3 の B/C を決める。
当たらない・ノイズだらけなら設計を見直す。

## 既知の割り切り（スパイクなので）
- 呼び出し元の「変更/未変更」判定は **呼び出し箇所の行が変更行集合に含まれるか**で近似（呼び出し元*関数*全体の変更判定はしない）
- LSP が Call Hierarchy 非対応の言語では呼び出し元が空になる
- 削除のみの hunk（新規行なし）は対象外
- リネーム・バイナリは素通し
